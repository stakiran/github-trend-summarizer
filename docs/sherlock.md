## Sherlock — ユーザー名からSNSアカウントを横断検索するCLIツール

### これは何？

**Sherlock** は、ひとつのユーザー名を入力するだけで **約490のSNS・Webサービス** を並列にスキャンし、「そのユーザー名でアカウントが存在するか」を一覧表示するPython製のコマンドラインツール。OSINT（公開情報に基づく調査）分野のデファクトツールのひとつ。

```bash
sherlock targetuser          # 単一ユーザー名
sherlock user1 user2 user3   # 複数同時検索
```

検出方法はサイトごとに3種類使い分けている:

| 検出方式 | 仕組み |
|---|---|
| **ステータスコード** | HTTPレスポンスコード(404等)で判定 |
| **エラーメッセージ** | HTML内の「Not Found」等の文字列の有無で判定 |
| **リダイレクトURL** | 存在しないユーザーがリダイレクトされるか否かで判定 |

---

### 何が嬉しいのか？ ― 既存手段との比較

| 観点 | 手動検索 / Google検索 | 類似ツール (Namechk, KnowEm等) | **Sherlock** |
|---|---|---|---|
| **対応サイト数** | 1つずつ手動 | 100〜300程度（Web UI） | **約490サイト**、JSON定義で容易に追加可 |
| **実行速度** | 極めて遅い | サーバ依存 | **20スレッド並列**で数十秒〜 |
| **コスト** | 無料 | 多くは有料プラン | **完全無料・OSS** |
| **自動化・組込み** | 不可 | API有料 | CLI/Docker/Python関数として自由に組込み可 |
| **偽陽性対策** | — | 少ない | WAF検知・除外リスト・正規表現バリデーション |
| **出力形式** | — | Web画面のみ | **TXT / CSV / Excel / ブラウザ自動表示** |

最大の強みは **「無料・ローカル実行・カスタマイズ可能・自動化可能」** の4点。セキュリティ調査やペネトレーションテストのワークフローに直接組み込める点が、Web UIベースの類似サービスと決定的に異なる。

### 使い方の流れ

```
┌─────────────────────────────────────────────────────┐
│ 1. インストール                                       │
│    $ pipx install sherlock-project                   │
│    （または docker / pip / OS パッケージマネージャ）      │
├─────────────────────────────────────────────────────┤
│ 2. ユーザー名を指定して実行                             │
│    $ sherlock johndoe                                │
│    → 約490サイトへ並列リクエストが走る（数十秒）          │
├─────────────────────────────────────────────────────┤
│ 3. 結果を確認                                         │
│    ターミナルに緑色でヒットしたサイトとURLが表示される      │
│    同時に johndoe.txt にURLリストが自動保存              │
├─────────────────────────────────────────────────────┤
│ 4. 必要に応じてオプションを活用                          │
│    --csv / --xlsx   → 構造化データとして出力             │
│    --site Instagram  → 特定サイトだけに絞る             │
│    --browse          → ヒットしたURLをブラウザで一括表示  │
│    --nsfw            → NSFWサイトも含める               │
│    --proxy socks5://… → Tor等のプロキシ経由で実行        │
│    --timeout 30      → タイムアウト秒数の調整            │
└─────────────────────────────────────────────────────┘
```

**典型的なユースケース:**
- **セキュリティ調査**: 対象人物のオンラインプレゼンスを網羅的に把握
- **自分のアカウント棚卸し**: 過去に作ったまま放置しているアカウントの発見
- **ブランド保護**: 自社名やブランド名の不正使用がないか確認
- **ユーザー名の空き確認**: 新規サービス登録前に、各所での使用状況を一括チェック
