## OmX (Oh My codeX) — リポジトリ概要

### これは何？

OpenAI の **Codex CLI** を拡張するワークフローオーケストレーション・レイヤー。Codex CLI そのものを置き換えるのではなく、「上に被せて使う」ツールキットで、以下の機能を追加する。

| 機能 | 概要 |
|---|---|
| **30+ エージェント定義** | architect / executor / debugger / reviewer 等の専門ロール。推論レベルやツール権限が個別設定 |
| **40 ワークフロースキル** | `$deep-interview`（要件明確化）、`$ralplan`（合意形成プランニング）、`$ralph`（完了まで繰り返すループ）、`$team`（並列実行）など |
| **チームモード** | tmux でペインを分割し、複数ワーカーが共有状態ファイルで協調。セッション切断後も再接続可能 |
| **HUD** | 現在のモード・タスクキュー・ワーカー状況をリアルタイム表示するダッシュボード |
| **MCP サーバー群** | 状態管理・プロジェクトメモリ・チーム通信・コード分析を Codex エージェントに提供 |

---

### 既存手段と比べて何が嬉しいのか？

| 比較対象 | OmX の優位点 |
|---|---|
| **素の Codex CLI** | Codex は 1 セッション 1 エージェント。OmX は tmux ベースで**複数エージェントの永続的な並列実行**と状態管理を追加する |
| **Claude Code のサブエージェント** | サブエージェントはインプロセスで完了を待つ一過性の存在。OmX のチームモードは**耐障害・再接続可能な durable セッション** |
| **手動プロンプトエンジニアリング** | `$skill-name` キーワードを入力に含めるだけで、**ソクラテス式ヒアリング → 計画策定 → 実装 → 検証**の定型パイプラインが自動起動する |
| **カスタムスクリプト / Makefile** | `.omx/` ディレクトリにプラン・メモリ・ログを永続化し、セッション横断で文脈を引き継ぐ。再利用性が高い |

要するに「**Codex CLI を "1 回限りの対話ツール" から "チーム開発可能な自律エージェント基盤" に格上げする**」のが主な価値。

### 使い方の流れ

```
1. インストール
   npm install -g @openai/codex oh-my-codex
   omx setup          # ~/.codex/ にプロンプト・スキル・AGENTS.md を配置
   omx doctor         # 環境チェック

2. 起動
   omx --madmax --high   # 高並列・高推論モードで Codex を起動

3. セッション内でスキルを呼び出す
   $deep-interview "何を作るか明確にしたい"
       → ソクラテス的にヒアリングし、要件を .omx/plans/ に PRD として保存

   $ralplan "計画を承認して"
       → 合意形成プランニング。テスト仕様まで自動生成

   $ralph "承認済み計画を実装して"
       → 完了検証を繰り返す永続ループ。失敗しても自動リトライ

   $team 3:executor "テストを並列で修正して"
       → tmux 3 ペインに executor ワーカーを起動、協調実行

4. モニタリング（別ターミナルで）
   omx hud --watch       # リアルタイムステータス
   omx team status <名前> # チーム進捗確認

5. 再接続・終了
   omx team resume <名前>    # 切断後に再接続
   omx team shutdown <名前>  # チーム終了
```

状態はすべて **`.omx/`** ディレクトリに永続化されるため、セッションをまたいだ作業の引き継ぎが可能。TypeScript + Rust のハイブリッド構成で、高速なコード探索（`omx explore`）やシェル検証（`omx sparkshell`）も Rust 側で提供している。
