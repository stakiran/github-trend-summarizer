## OpenScreen — リポジトリ概要

### これは何？

**OpenScreen** は、プロダクトデモやチュートリアル動画を作成するための **Electron 製デスクトップアプリ**（v1.3.0）。画面録画 → 編集 → エクスポートの一連の流れをワンストップで提供する。

技術スタックは **React 18 + TypeScript + Vite + Tailwind CSS** のフロントエンドに、映像処理として **WebCodecs API / MediaRecorder / PixiJS（WebGL）/ gif.js** を組み合わせている。macOS・Windows・Linux のクロスプラットフォーム対応。

---

### 何が嬉しいのか？（既存ツールとの比較）

| 観点 | **Screen Studio**（有料） | **OBS + 別エディタ** | **OpenScreen** |
|---|---|---|---|
| 価格 | $89〜（買い切り） | 無料だが手間大 | **完全無料・OSS（MIT）** |
| ウォーターマーク | なし | なし | **なし** |
| 商用利用 | ライセンス次第 | OK | **OK** |
| 録画→編集の一貫性 | ○ 一体型 | ✗ ツール分断 | **○ 一体型** |
| ズーム・モーションブラー | ○ | 手動で後編集 | **○（自動カーソル追従ズーム対応）** |
| アノテーション（矢印・テキスト） | ○ | 別ツール必要 | **○** |
| GIF エクスポート | ○ | 別ツール必要 | **○（15-30 FPS, 解像度プリセット）** |
| プラットフォーム | macOS のみ | 全 OS | **全 OS（macOS/Win/Linux）** |

**要するに**、Screen Studio と同等の「画面録画 + おしゃれな編集」体験を、**無料・OSS・クロスプラットフォーム**で実現する点が最大の差別化ポイント。OBS のように録画と編集が分断されず、背景の壁紙・影・ズームアニメーションなどの演出を GUI 上で直感的に付与できる。

### 使い方の流れ

```
① 起動  →  ② ソース選択  →  ③ 録画  →  ④ 編集  →  ⑤ エクスポート
```

1. **起動** — アプリを起動すると、画面上部に小さな **HUD オーバーレイ**（常時最前面）が表示される。マイク・システム音声・Web カメラの ON/OFF をここで切り替える。

2. **ソース選択** — 「ソース選択」ボタンを押すと、利用可能なスクリーンやウィンドウがサムネイル付きで一覧表示される。対象をクリックして選択。

3. **録画** — HUD の録画ボタンを押すと、選択したソースの録画が開始。WebM 形式で `userData/recordings/` に保存される。最大 4K・60FPS 対応。コーデックは AV1 → H.264 → VP9 の優先順位で自動選択。

4. **編集** — 録画停止と同時にエディタウィンドウが開く。タイムライン上で以下を操作できる：
   - **ズーム領域** — 手動 or カーソル追従で注目箇所を拡大（6段階）
   - **トリム** — 不要部分のカット
   - **速度変更** — 0.25x〜2x の可変速
   - **アノテーション** — テキスト・画像・矢印の追加（PixiJS レンダリング）
   - **背景** — 18種の壁紙、グラデーション、単色から選択
   - **影・モーションブラー** — 強度スライダーで調整
   - **クロップ** — 表示領域の切り抜き
   - Undo/Redo 対応、`.openscreen` 形式でプロジェクト保存・再開も可能

5. **エクスポート** — MP4（Medium/Good/Source品質）または GIF を選択し、任意のパスに書き出す。WebCodecs API によるハードウェアアクセラレーション対応。

### 補足：セットアップ

```bash
# 開発環境
npm install && npm run dev

# ビルド（配布用）
npm run build   # → release/ にDMG/NSIS/AppImage生成
```

macOS では初回起動時に「画面収録」と「アクセシビリティ」の権限付与が必要。Linux では PipeWire がシステム音声キャプチャの前提条件となる。
