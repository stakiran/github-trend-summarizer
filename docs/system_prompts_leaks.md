## このリポジトリは何？

**主要AIモデルのシステムプロンプト（内部指示文）を収集・アーカイブしたリポジトリ。** 各AIサービスが裏でモデルに渡している「振る舞いの指示書」をMarkdownファイルとして128件以上蓄積している。対象は以下の通り：

| ベンダー | 主な収録モデル |
|---|---|
| **OpenAI** | GPT-5.4 / 5.3 / 5.2 / 5.1, Codex CLI, o3, o4-mini, ツール定義・安全ポリシー |
| **Anthropic** | Claude Opus 4.6 / Sonnet 4.6, Claude Code, Cowork, Chrome拡張, Excel |
| **Google** | Gemini 3.1 Pro / 3 Flash, Gemini CLI, Jules, NotebookLM |
| **xAI** | Grok 4.2 / 4 / 3, ペルソナ, 安全指示 |
| **その他** | Perplexity, Notion AI, Copilot (Word), Kagi, Le Chat, Raycast AI 等 |

各プロンプトは**編集・要約なしの生テキスト**で保存され、旧バージョンは `old/` フォルダに、未加工の生データは `raw/` フォルダに整理されている。

## 何が嬉しいのか？（既存手段との比較）

| 比較軸 | 本リポジトリ | 既存の手段 |
|---|---|---|
| **網羅性** | 5社・30モデル超を一箇所に集約。バージョン別・ツール有無別の差分もある | 個別のブログ記事やSNS投稿に散在しており、横断比較が困難 |
| **鮮度** | 定期更新。GPT-5.4やClaude 4.6など最新モデルまでカバー | スナップショット的な単発記事が多く、すぐ古くなる |
| **信頼性** | 生テキストをそのまま掲載し改変なし。PRベースで第三者検証も入る | 個人ブログ等では抜粋・意訳されていることが多い |
| **比較しやすさ** | 同一ディレクトリ構造でベンダー横断比較が容易（例：安全ポリシーの違い） | 別々のソースから自力で収集・整形する必要がある |

**主なユースケース：**
- **プロンプトエンジニアリングの参考**：プロが書いたシステムプロンプトの構造・テクニックを学べる
- **AIモデル間の設計思想比較**：安全ポリシー、ツール定義、パーソナリティ設計の違いを横断的に分析
- **バージョン間の変化追跡**：同一モデルのプロンプトがどう進化しているかをdiffで確認

## 使い方の流れ

```
1. リポジトリをクローン or GitHub上で閲覧
   $ git clone https://github.com/asgeirtj/system_prompts_leaks.git

2. ベンダー別フォルダから目的のプロンプトを開く
   Anthropic/ → Claude系
   OpenAI/   → ChatGPT/GPT系
   Google/   → Gemini系
   xAI/      → Grok系
   Misc/     → その他

3. 必要に応じて比較・分析
   - 同モデルのツール有/無版を比較（例: claude-opus-4.6.md vs claude-opus-4.6-no-tools.md）
   - old/ フォルダで旧バージョンとのdiffを取る
   - 複数ベンダーの安全ポリシーを横断比較

4. 貢献する場合
   - 新しいプロンプトを .md ファイルとして該当フォルダに配置 → PR提出
   - リクエストがあれば Issue を立てる
```
