## Claudian — 概要整理

### これは何？

Obsidian のコミュニティプラグイン。**Claude Code（および Codex）のエージェントランタイムを Obsidian のサイドバーに直接埋め込む**もの。Vault ディレクトリがそのままエージェントの作業ディレクトリになり、チャット UI からファイルの読み書き・検索・Bash 実行・マルチステップ作業をすべて Obsidian 内で完結できる。

内部的には Claude Code CLI / Codex CLI を子プロセスとして起動し、公式 SDK のストリーミング出力をパースして Obsidian UI に投影するアダプター構造を取っている。

---

### 何が嬉しいのか？（既存手段との比較）

| 比較軸 | ターミナルで Claude Code を直接使う | Obsidian Copilot 等の汎用 AI プラグイン | **Claudian** |
|---|---|---|---|
| **Vault 統合** | Vault を cwd にすれば可能だが UI は別窓 | Obsidian 内だがファイル操作は限定的 | Vault がそのまま作業対象。`@` でファイル・ディレクトリを直接メンション |
| **エージェント能力** | フル（ファイル編集・Bash・MCP 等） | API 呼び出し中心、ツール利用は限定的 | Claude Code のフル能力をそのまま利用（ツール実行・Plan モード・MCP サーバー接続） |
| **インライン編集** | なし（差分はターミナル表示） | 一部プラグインが対応 | ノート上でテキスト選択 → ホットキーで **単語レベルの差分プレビュー付き編集** |
| **会話管理** | `.jsonl` トランスクリプト | 単発チャットが多い | マルチタブ・履歴・フォーク・リワインド・コンパクト化を UI で操作 |
| **拡張性** | スラッシュコマンド・MCP | プラグイン依存 | `/` コマンド、`$` スキル、`#` インストラクション、MCP サーバー、サブエージェント（`@agent`）をすべてVaultスコープで管理 |

**一言で言うと**：ターミナルに切り替えずに Claude Code のフルパワーを Obsidian の中で使える点と、Vault のコンテキスト（ファイル・ノート・ディレクトリ構造）をシームレスに渡せる点が最大の差別化要素。

### 使うときの流れ

```
1. 前提準備
   ├─ Claude Code CLI をインストール（npm or ネイティブ）
   ├─ Claude のサブスクリプション or API キーを設定済みにしておく
   └─ （任意）Codex CLI もインストール

2. プラグイン導入
   ├─ GitHub Releases から手動 DL → .obsidian/plugins/claudian/ に配置
   ├─ または BRAT 経由で自動アップデート管理
   └─ Obsidian の設定 → コミュニティプラグイン → Claudian を有効化

3. 初期設定（Settings → Claudian）
   ├─ CLI パスの指定（nvm 等を使っている場合）
   ├─ プロバイダー選択（Claude / Codex）
   └─ 環境変数・権限の調整

4. 日常利用
   ├─ サイドバーのチャットパネルからプロンプトを送信
   ├─ @ファイル名 でコンテキストを渡す
   ├─ Shift+Tab で Plan モードに切り替え → 実装前に設計を確認
   ├─ テキスト選択 → ホットキーでインライン編集（差分プレビュー→承認）
   ├─ /コマンド や $スキル で定型操作を呼び出し
   └─ 会話のフォーク・リワインド・タブ切替で複数作業を並行管理
```
